Modelo Almacen de Maderas

Modelo Entidad-Relacion

Se ha disenado el modelo E/R de un almacen de maderas donde se representan clientes, pedidos, productos y proveedores.

Este modelo permite gestionar tanto las ventas como las compras de materiales, asi como el control de stock.

Captura ABD28: Modelo E/R del sistema.

Tablas creadas

Clientes

  • id_cliente
  • nombre
  • telefono
  • ciudad

Pedidos

  • id_pedido
  • fecha
  • total
  • id_cliente (FK)
  • id_empleado (FK)

Empleados

  • id_empleado
  • nombre
  • puesto
  • salario

Productos

  • id_producto
  • nombre
  • precio_m3
  • stock
  • id_tipo (FK)

Tipos_Madera

  • id_tipo
  • nombre
  • densidad
  • origen

Proveedores

  • id_proveedor
  • nombre
  • telefono
  • ciudad

Compras

  • id_compra
  • cantidad
  • fecha
  • id_proveedor (FK)
  • id_producto (FK)

Detalle_Pedido

  • id_detalle
  • cantidad
  • precio_unitario
  • id_pedido (FK)
  • id_producto (FK)

Relaciones del modelo

  • Clientes - Pedidos (1:N)
  • Empleados - Pedidos (1:N)
  • Pedidos - Detalle_Pedido (1:N)
  • Productos - Detalle_Pedido (1:N)
  • Tipos_Madera - Productos (1:N)
  • Proveedores - Compras (1:N)
  • Productos - Compras (1:N)

Relaciones N:M:

  • Pedidos y Productos mediante Detalle_Pedido
  • Proveedores y Productos mediante Compras

Conclusion
