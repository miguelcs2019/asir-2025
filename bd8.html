Modelo Almacen de Maderas

Modelo Entidad-Relacion

En esta actividad se ha disenado el modelo Entidad-Relacion correspondiente a un sistema de gestion de un almacen de maderas.

El modelo representa clientes, pedidos, productos, proveedores y el flujo completo de compras y ventas dentro del almacen.

Captura ABD26: Modelo E/R del almacen de maderas.

Tablas creadas (Entidades)

Se han creado las siguientes tablas en la base de datos:

  • Clientes: id_cliente, nombre, telefono, ciudad
  • Pedidos: id_pedido, fecha, total, id_cliente, id_empleado
  • Empleados: id_empleado, nombre, puesto, salario
  • Productos: id_producto, nombre, precio_m3, stock, id_tipo
  • Tipos_Madera: id_tipo, nombre, densidad, origen
  • Proveedores: id_proveedor, nombre, telefono, ciudad
  • Compras: id_compra, cantidad, fecha, id_proveedor, id_producto
  • Detalle_Pedido: id_detalle, cantidad, precio_unitario, id_pedido, id_producto

Relaciones del modelo

  • Clientes – Pedidos (1:N): un cliente puede realizar varios pedidos
  • Empleados – Pedidos (1:N): un empleado gestiona varios pedidos
  • Pedidos – Detalle_Pedido (1:N): un pedido tiene varias lineas
  • Productos – Detalle_Pedido (1:N): un producto puede aparecer en varios pedidos
  • Tipos_Madera – Productos (1:N): un tipo de madera tiene varios productos
  • Proveedores – Compras (1:N): un proveedor realiza varias compras
  • Productos – Compras (1:N): un producto puede comprarse varias veces

Estas relaciones generan dos relaciones de tipo N:M:

  • Pedidos y Productos mediante la tabla Detalle_Pedido
  • Proveedores y Productos mediante la tabla Compras
