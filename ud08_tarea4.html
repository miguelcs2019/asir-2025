UD08 – Tarea 4: Configuración de RAID 0 por Software

Objetivo de la práctica

En esta práctica se configura un RAID 0 por software utilizando mdadm. El objetivo es combinar dos discos en un único volumen lógico, aumentando el rendimiento mediante distribución de bloques (striping).

El volumen final se montará en /srv/datosExtendidos y deberá configurarse para montarse automáticamente tras el reinicio del sistema.

2.0 – Añadido de discos en Proxmox

Se añaden dos discos virtuales adicionales de 10GB a la máquina virtual. Estos discos serán utilizados para crear el RAID 0.

Captura 2.0: Discos añadidos en el entorno Proxmox.

2.1 – Verificación de detección en Linux

Mediante el comando lsblk se verifica que el sistema detecta correctamente los nuevos discos (por ejemplo /dev/sdb y /dev/sdc).

Captura 2.1: Detección de discos adicionales en Linux.

2.2 – Particionado de los discos

Se crea una partición primaria en cada disco ocupando todo el espacio disponible. Posteriormente se asigna el tipo Linux raid autodetect (fd).

Captura 2.2: Particiones preparadas para RAID.

2.3 – Creación del RAID 0

Se crea el array RAID 0 mediante el comando: mdadm --create /dev/md0 --level=raid0 --raid-devices=2 /dev/sdb1 /dev/sdc1.

El comando cat /proc/mdstat confirma que el array se encuentra activo.

Captura 2.3: RAID 0 activo en /dev/md0.

2.4 – Creación del sistema de archivos

Se crea un sistema de archivos ext4 sobre el dispositivo /dev/md0 utilizando mkfs.ext4.

Posteriormente se verifica con lsblk -f que el volumen tiene el sistema de archivos correctamente asignado.

Captura 2.4: Sistema de archivos ext4 creado en RAID 0.

2.5 – Montaje del RAID 0

Se crea el punto de montaje /srv/datosExtendidos y se monta manualmente el volumen.

El comando df -h confirma que el volumen aparece montado correctamente y muestra un tamaño aproximado de 20GB, resultado de la suma de ambos discos.

Captura 2.5: RAID 0 montado en /srv/datosExtendidos.

2.6 – Configuración automática y verificación tras reinicio

Se añade la definición del array en /etc/mdadm/mdadm.conf, se actualiza el initramfs y se configura el montaje automático en /etc/fstab utilizando el UUID del volumen.

Tras reiniciar el sistema, el comando df -h confirma que /dev/md0 se monta automáticamente en /srv/datosExtendidos.

Captura 2.6: Verificación del montaje automático tras reinicio.

Conclusión

El RAID 0 ha sido configurado correctamente. Se ha comprobado su funcionamiento y su montaje automático tras reinicio. El volumen final presenta un tamaño equivalente a la suma de los discos utilizados, lo que confirma el comportamiento característico del RAID 0.

A diferencia del RAID 1, el RAID 0 mejora el rendimiento pero no ofrece tolerancia a fallos, ya que la pérdida de uno de los discos implica la pérdida total de los datos.
