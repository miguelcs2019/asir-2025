27.8 – Script ultimas 10 entradas de programas

#!/bin/bash
sudo journalctl _COMM=scite -n 10
sudo journalctl _COMM=firefox -n 10
sudo journalctl -u NetworkManager -n 10
sudo journalctl _COMM=sensors -n 10

27.9 – Logs del usuario root

sudo journalctl _UID=0 -n 20

27.10 – Script logs por grupo

#!/bin/bash
grupo=$1
for usuario in $(getent group $grupo | cut -d: -f4 | tr ',' ' ')
do
    uid=$(id -u $usuario)
    sudo journalctl _UID=$uid -n 10
done

27.11 – NetworkManager ultimas 36 horas

sudo journalctl -u NetworkManager --since "36 hours ago" --no-pager

27.12 – NetworkManager ultima semana

sudo journalctl -u NetworkManager --since "7 days ago" --no-pager

27.13 – Logs del usuario miguelc ayer

sudo journalctl _UID=$(id -u miguelc) --since "yesterday" --until "today" --no-pager

El mensaje -- No entries -- indica que no hubo eventos registrados para ese usuario en el rango temporal indicado.

27.15 – Funcionamiento real del visor grafico

Se reinicia el servicio NetworkManager para generar actividad: 

sudo systemctl restart NetworkManager

Posteriormente se abre GNOME Logs, se filtra por "NetworkManager" y se visualizan los eventos generados.

Manual de uso – GNOME Logs

1. Instalacion

sudo apt install gnome-logs

2. Ejecucion

gnome-logs

3. Interfaz

  • Panel lateral con categorias (Sistema, Aplicaciones, Seguridad).
  • Zona central con eventos cronologicos.
  • Barra de busqueda superior.

4. Filtrado

  • Busqueda por palabra clave.
  • Filtrado por servicio.
  • Clasificacion por prioridad.
  • Consulta de eventos recientes.

5. Aplicacion practica

  • Diagnostico rapido de errores.
  • Supervision de servicios.
  • Analisis de fallos de red.
  • Auditoria basica del sistema.

La herramienta ofrece una alternativa grafica a journalctl, facilitando la interpretacion de eventos sin necesidad de trabajar exclusivamente en terminal.
